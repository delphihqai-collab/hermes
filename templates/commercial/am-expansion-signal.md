# AM Expansion Signal

Used by: AM → HERMES
Trigger: When AM identifies a credible upsell or expansion opportunity
Rule: AM flags and stops. Never pitches directly. HERMES decides if and when to engage commercially.
Prerequisite: Client must have demonstrated clear value from their current contract before any expansion is considered.

```
EXPANSION SIGNAL — [Client Name]
──────────────────────────────────
Client:              [Name, company, sector]
Date identified:     [DD/MM/YYYY]
Current contract:    [Scope summary, renewal date]
Health score:        [Healthy / At Risk — expansion only progresses from Healthy]

── THE SIGNAL ──────────────────────────
What was observed:   [Specific — what the client said, did, or what data showed]
Source:              [Check-in call / usage data / client message / sector news]
Client's exact words: [If verbal — quote directly]

── EXPANSION OPPORTUNITY ───────────────
Type:                [New scope / Additional location / More users / New workflow / Other]
Description:         [What additional value Delphi could deliver — be specific]
Delphi capability:   [Can we deliver this today? Y/N — if N, flag to HERMES for product discussion]
Estimated fit:       [Strong / Moderate / Speculative — and why]

── CONTEXT ─────────────────────────────
Value already delivered: [Evidence that the client is getting ROI from current contract]
Relationship quality:  [Strong / Neutral / Recovering — relevant to timing]
Right time to engage:  [Y/N — and why. If N, suggest when to revisit]

Flagging to HERMES for commercial decision. AM will not raise this with the client without explicit approval.
```
