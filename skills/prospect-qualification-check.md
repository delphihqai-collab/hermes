# Prospect Qualification Check

**Type:** Decision routine
**Applies to:** Evaluating whether a prospect should advance from SDR to AE
**Trigger keywords:** qualify prospect, SDR handoff, ready for AE, is this worth AE time, qualify lead, prospect fit, ICP check, handoff decision
**Times used:** 0
**Last used:** —

## Skill

Run this check before any SDR-to-AE handoff. If a prospect fails, they go back to SDR for nurture or are disqualified. Do not let bad fits consume AE time.

**CHECK 1 — ICP FIT**
Does this prospect match Delphi's Ideal Customer Profile?
[ASK PEDRO — document ICP criteria here: industry, size, role of buyer, geography]
Pass / Fail / Partial (note why)

**CHECK 2 — PROBLEM FIT**
Has the prospect expressed a problem that Delphi solves? Not "they might have this problem." They have articulated it, or SDR has confirmed it.
Pass / Fail

**CHECK 3 — ACCESS TO DECISION-MAKER**
Is the contact a decision-maker or do they have confirmed access to one?
Pass / Fail / Unknown (if unknown, SDR must clarify before handoff)

**CHECK 4 — TIMING**
Is there any indication of active buying intent or a timeline?
Pass / Fail / Unknown (acceptable to pass with Unknown if checks 1-3 are strong)

**CHECK 5 — NO BLOCKERS**
No open legal, compliance, or relationship issues with this account.
Pass / Fail

**RESULT:**
- 5/5 Pass → Handoff approved, use SDR handoff template
- 4/5 Pass (Check 4 Unknown acceptable) → Handoff approved with note
- Any Fail on Checks 1, 2, 3, or 5 → Return to SDR with specific gap to address
- Fail on 2+ checks → Disqualify or long-term nurture, document reason
