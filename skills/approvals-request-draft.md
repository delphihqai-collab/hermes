## Skill

Load the approvals-gate playbook first. Then build the approval request using this process:

**STEP 1 — Verify the draft is final**
Do not submit a draft that still needs work. The approval is for exactly what is submitted. If the underlying draft is not ready, finish it first.

**STEP 2 — Write the ACTION line**
"Send [type of message] to [recipient] regarding [topic]"
Example: "Send discovery call follow-up to João Silva at Coverflex re: proposal timing"

**STEP 3 — Write CONTEXT in two sentences**
Sentence 1: Where are we in the relationship / deal?
Sentence 2: Why is this being sent right now?

**STEP 4 — Paste the exact draft**
No placeholders. No [INSERT NAME]. The actual final content.

**STEP 5 — Write RISK IF DELAYED honestly**
If there is genuine urgency, state it plainly. If there is no urgency, write "No critical time pressure." Never manufacture urgency to get faster approval.

**STEP 6 — Post to #approvals and note it in the daily session log**

## Notes

- Keep the approval request itself short — the draft is the main content
- If Boss approves with modifications, treat the modified version as a new draft requiring confirmation before sending
